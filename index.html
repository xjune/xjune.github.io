Create Blog

背景

为什么采用github？[from 阮一峰]

  • 第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
  • 第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
  • 第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

为什么采用hexo框架？［from sunwhut］

  • 极速生成静态页面
  • 支持Markdown
  • 一键部署博客
  • 丰富的插件支持

本地环境搭建

本文命令基于mac os，windows用户采用教程：

  • sunwhut
  • 石金融

####前提

  • Node.js
  • Git
  • GitHub Desktop (可选)
  • Xcode（mac os appstore）

####安装Hexo
npm install hexo -g
安装完成后，在任意目录下创建子目录，即为本地博客目录：

cd ~ #这里就是你的任意目录，~为用户根目录
mkdir MyBlog #可任意命名
cd MyBlog #打开博客目录

执行以下命令初始化本地博客：

hexo init
npm install
hexo s #启动本地服务器测试, 查看样式

打开浏览器，输入http://localhost:4000,查看本地网站样式。

####上传博文
hexo new post “article title”
这条命令会在你的博客目录的 source/_posts 下生成一个 article-title.md 的文件，用编辑器打开就可以编辑文章了。

Hexo 使用的 Markdown 语法，这里推荐几种编写 Markdown 的编辑器：
Window: MarkdownPad2
Mac: Mou
Linux: ReText
多平台：Sublime Text + MarkdownEditting(插件)

文章编辑好之后，运行生成、部署命令：

$ hexo g   // 生成
$ hexo d   // 部署

在浏览器中刷新博客，就可以看到，由于 Github 的缓存机制，可能要多刷新几次才能看到。

####删除博文

There is no command to delete a post on Hexo, but follow this steps :

  1. Delete the post under source/_post folder
  2. Run hexo clean to delete the database (db.json) and assets folder
  3. Run hexo generate to generate the new blog without your deleted post
  4. Run hexo deploy to deploy your blog

Alternatively you can use the hexo-admin plugin.

Coursera ML 1

ML lecture 1

线性回归模型

Model and Representation

  • hypothesis

    模型：输入数据－》h(x)－》输出预测

    1. Linear regression

      $$h_\theta(x) = \theta_0 + \theta_1x$$

      shorthand:h(x)

  • cost function

    代价函数：

    $$minimize_{\theta_0\theta_1}(\sum\limits_{i=1}^m (h_{\theta}(x^i) - y^i)^2)$$

    m:number of training set
    contour plot描述代价函数

Minimize cost function

Gradient descent

梯度下降：获得局部最优解
  • Algorithm

repeat until convergence ｛

$$\theta_j := \theta_j - \alpha \frac\partial{\partial\theta_j}J(\theta_0,\theta_1)$$
(for j = 0 and j = 1)

｝

\(\alpha\):learning rate（学习速率）

Correct: simultaneous update

$$temp0 := \theta_0 - \alpha\frac{\partial}{\partial\theta_0}J(\theta_0, \theta_1)$$

$$temp1 := \theta_1 - \alpha\frac{\partial}{\partial\theta_1}J(\theta_0, \theta_1)$$

$$\theta_0 := temp0$$

$$\theta_1 := temp1$$

Incorrect

$$temp0 := \theta_0 - \alpha\frac{\partial}{\partial\theta_0}J(\theta_0, \theta_1)$$

$$\theta_0 := temp0$$

$$temp1 := \theta_1 - \alpha\frac{\partial}{\partial\theta_1}J(\theta_0, \theta_1)$$

$$\theta_1 := temp1$$

为什么梯度下降算法需要同步更新？

  • 直观体验梯度下降算法

    \(\alpha\) (learning rate)过小，下降速度太慢，耗时久。过大，难以收敛，甚至发散。

    当到达偏导数为0的点，参数值将不会在改变。此时位于局部最低点（而不一定为全局最小）

    偏导数的目的是为了取方向。参数向导数反向移动，逼近导数为0的点

    凸函数(convex function)只有一个最低点，所以使用梯度下降算法可以到达全局最低点。

Conclusion

Hypothesis

$$h_\theta(x) = \theta_0 + \theta_1 x$$

Parameters

$$\theta_0 , \theta_1$$

Cost Function

$$J(\theta_0 , \theta_1) = \frac1{2m} \sum\limits_{i=1}^m(h_{\theta}(x^i)-y^i)^2$$

Goal

$$minimize_{\theta_0\theta_1} J(\theta_0 , \theta_1)$$

Algorithm

GD 求取 Goal

“Batch” Gradient Descent
Batch： Each step of gradient descent uses all the training examples.

以上为运用梯度算法的线性回归

也可使用正规方程(normal equations)

数据量较大的情况下，梯度算法比正规方程更适用
